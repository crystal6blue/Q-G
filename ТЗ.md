# Техническое задание: AI Kahoot - Генератор тестовых вопросов

## 1. Описание проекта

Веб-приложение для автоматической генерации тестовых вопросов с использованием LLM (Ollama). Система затем сохраняет вопросы в БД для дальнейшего использования. Аналог Kahoot с фокусом на генерацию контента.

---

## 2. Основной функционал

### 2.1 Генерация вопросов
- **Из текстового промпта** - пользователь передает тему, система генерирует вопросы
- **Из URL** - система загружает контент со ссылки (Wikipedia, статьи) и генерирует вопросы
- **Из файла** - поддержка PDF, DOCX, TXT, MD файлов

### 2.2 Структура вопроса
```json
{
  "question": "текст вопроса",
  "variants": ["ответ1", "ответ2", "ответ3", "ответ4"],
  "correct_answer": "ответ1",
  "explanation": "почему это верный ответ",
  "hint": "подсказка для ученика",
  "category": "тема/предмет",
  "confidence_score": 0.95
}
```

### 2.3 Управление квизами
- **Сохранение** - сохранение сгенерированных или вручную созданных квизов
- **Просмотр** - получение сохраненного квиза по ID
- **Список** - вывод всех квизов с фильтрацией по сложности
- **Удаление** - удаление квиза и всех связанных вопросов

## 3. Технологический стек

| Компонент | Технология | Назначение |
|-----------|-----------|-----------|
| **Backend** | FastAPI 0.115 | REST API |
| **ORM** | SQLAlchemy 2.0 | Работа с БД |
| **БД** | PostgreSQL 16 | Хранение квизов и вопросов |
| **LLM** | Ollama (qwen2.5:0.5b) | Генерация вопросов |
| **HTTP Client** | httpx | Запросы к URLs, Ollama API |
| **Парсинг** | BeautifulSoup4 | Извлечение текста из HTML |
| **Контейнеризация** | Docker Compose | Оркестрация сервисов |

---

## 4. API Endpoints

### 4.1 Генерация вопросов

```bash
POST /questions/prompt
POST /questions/from-url
POST /questions/from-file
```
**Response:** Список вопросов + модель LLM + raw output (опционально)

### 4.2 Сохранение квизов

```bash
POST /quizzes/save              # Сохранить ручной киз
POST /questions/prompt/save     # Сгенерировать и сохранить
```
**Response:** ID квиза + сообщение о успехе

### 4.3 Получение квизов

```bash
GET /quizzes                    # Список всех квизов (с фильтрацией и пагинацией)
GET /quizzes/{id}               # Получить конкретный киз
DELETE /quizzes/{id}            # Удалить киз
```

### 4.4 健康проверка

```bash
GET /health                     # Проверка статуса API
```

## 5. База данных

### 5.1 Таблица `quizzes`
- `id` - primary key
- `title` - название (до 255 символов)
- `description` - описание
- `difficulty` - уровень (easy/medium/difficult)
- `question_style` - стиль (scientific/humanistic/general)
- `prompt` - исходный промпт
- `question_count` - количество вопросов
- `variant_count` - количество вариантов ответа
- `model` - используемая LLM модель
- `created_at` - timestamp создания

### 5.2 Таблица `questions`
- `id` - primary key
- `quiz_id` - foreign key на quizzes
- `question_text` - текст вопроса
- `variants` - JSON массив вариантов
- `correct_answer` - правильный ответ
- `explanation` - объяснение ответа
- `hint` - подсказка
- `category` - категория/тема
- `confidence_score` - уверенность модели (0-1)
- `created_at` - timestamp создания

## 6. Параметры генерации

| Параметр | Тип | Диапазон | По умолчанию | Описание |
|----------|-----|----------|--------------|---------|
| `prompt` | string | - | - | Тема/контекст для генерации |
| `question_count` | int | 1-50 | 5 | Количество вопросов |
| `variant_count` | int | 2-10 | 4 | Вариантов ответа на вопрос |
| `difficulty` | enum | easy/medium/difficult | medium | Уровень сложности |
| `question_style` | enum | scientific/humanistic/general | general | Стиль вопросов |

## 8. Требования к окружению

### 8.1 Обязательно
- Docker & Docker Compose
- 4+ GB RAM для контейнеров
- 10+ GB свободного места (для моделей и БД)

### 8.2 Порты
- **8000** - FastAPI
- **5432** - PostgreSQL
- **11434** - Ollama

### 8.3 Переменные окружения
```
DATABASE_URL=postgresql://quiz_user:quiz_password@postgres:5432/quiz_db
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=qwen2.5:0.5b
INCLUDE_RAW=0  # Включить raw output от модели (0/1)
```

## 11. Примеры использования

### Генерация из промпта
```bash
curl -X POST http://localhost:8000/questions/prompt \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "Python асинхронное программирование",
    "question_count": 3,
    "variant_count": 4,
    "difficulty": "medium",
    "question_style": "general"
  }'
```

### Сохранение квиза
```bash
curl -X POST http://localhost:8000/questions/prompt/save \
  -H "Content-Type: application/json" \
  -d '{
    "prompt": "FastAPI",
    "question_count": 2,
    "variant_count": 4,
    "difficulty": "medium",
    "question_style": "general"
  }'
```

### Получение квизов
```bash
curl http://localhost:8000/quizzes?difficulty=medium&skip=0&limit=10
```

## 13. Notes

- Модель LLM работает на **CPU** (без GPU) - достаточно для демо
- PostgreSQL автоматически инициализируется при первом запуске
- Ollama автоматически скачивает модель qwen2.5:0.5b (~200MB)
- All timestamps в UTC
